API Films

Routes publiques

Films

GET /api/films : https://cadriel-front.onrender.com/films

Retourne la liste de tous les films. Vous pouvez trier les films par année, par titre et par réalisateur de façon ascendante ou descendante à l'aide de paramèetres dans l'URL. Par exemple : /api/films?tri=annee&ordre=asc

GET /api/films/:id : https://cadriel-front.onrender.com/films/idFilm

Retourne le film ayant l'identifiant :id

POST /api/films : https://cadriel-front.onrender.com/films

Ajoute un nouveau film à la liste de films. Le corps de la requête doit contenir les informations du film à ajouter. Le film doit être ajouté à la base de données et l'identifiant du film doit être retourné.

PUT /api/films/:id : https://cadriel-front.onrender.com/films/idFilm

Modifier le film ayant l'identifiant :id. Le corps de la requête doit contenir les informations du films à modifier. Le film doit être modifié dans la base de données.

DELETE /api/films/:id : https://cadriel-front.onrender.com/films/idFilm

Supprime le film ayant l'identifiant :id de la base de données.

Utilisateurs

POST /api/utilisateurs/inscription :

Ajoute un nouvel utilisateur à la liste des utilisateurs. Le corps de la requête doit contenir les informations de l'utilisateur à ajouter. Le mot de passe n'a pas besoin d'être encrypté avant d'être ajouté à la base de données. L'identifiant de l'utilisateur doit être retourné. Assurez-vous de ne pas créer deux utilisateurs aec le même courriel. Retournez un false avec un messge d'erreur dans ce cas précis.

POST /api/utilisateurs/connexion :

Connecte un utilisateur. Le corps de la requête doit contenir les informations de l'utilisateur à connecter. Le mot de passe doit être comparé avec celui dans la base de données. Si le mot de passe est valide, votre API retournera true sinon elle retournera false avec un message d'erreur approprié.
